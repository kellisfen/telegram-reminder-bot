# Telegram-бот напоминалка для клиентов

Telegram-бот для хранения данных по договорам и автоматических напоминаний перед окончанием срока. Интеграция с Google Sheets как базой клиентов.

## Возможности

- Хранение данных о договорах в Google Sheets
- Добавление клиентов через бота (админом или самим клиентом)
- Автоматический расчёт даты окончания и даты напоминания
- Напоминания за 2 месяца до конца договора
- Проверка на дубликаты
- Отслеживание кто запускал бота (клиент или админ)

## Быстрый старт

### 1. Клонирование

```bash
git clone https://github.com/kellisfen/telegram-reminder-bot.git
cd telegram-reminder-bot
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка

```bash
cp .env.example .env
# Заполни .env своими значениями
```

**Получение Telegram токена:**
1. Напиши [@BotFather](https://t.me/BotFather) в Telegram
2. Создай бота: `/newbot`
3. Скопируй токен в `.env` → `TELEGRAM_BOT_TOKEN`

**Настройка Google Sheets:**
1. Создай проект в [Google Cloud Console](https://console.cloud.google.com/)
2. Включи Google Sheets API
3. Создай Service Account и скачай JSON ключ → `credentials.json`
4. Скопируй ID таблицы из URL → `SPREADSHEET_ID`
5. Расшарь таблицу для service account email

### 4. Запуск

```bash
python3 bot.py
```

## Структура проекта

```
├── bot.py              # Точка входа
├── config.py            # Конфигурация
├── scheduler.py         # Шедулер напоминаний
├── handlers/
│   ├── common.py        # /start, /help
│   ├── admin.py         # /clients, /stats
│   ├── client.py        # Добавление клиента
│   └── states.py        # FSM состояния
├── sheets/
│   └── client.py        # Google Sheets API
└── db/
    └── state.py         # SQLite — состояние клиентов
```

## Команды бота

| Команда | Описание |
|---------|---------|
| `/start` | Начало работы |
| `/help` | Помощь |
| `/clients` | Список клиентов (админ) |
| `/stats` | Статистика (админ) |
| `/check_dups` | Проверить дубли (админ) |

## Поля в Google Sheets

| Колонка | Описание |
|---------|----------|
| A | ID записи |
| B | Дата создания |
| C | Кто создал (admin/client) |
| D | Telegram username |
| E | Telegram ID |
| F | Контакт для ручного поиска |
| G | Дата начала договора |
| H | Срок (месяцев) |
| I | Дата окончания |
| J | Дата напоминания |
| K | Статус |
| L | Признак дубля |
| M | Комментарий по дублю |

## Логика напоминаний

- **Админ добавил клиента** → напоминание только админу
- **Клиент сам стартанул бота** → напоминание клиенту и админу
- **Клиент не запускал бота** → напоминание только админу
